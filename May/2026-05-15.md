### Today I Learned
#### 객체지향
- **객체지향**: 프로그램을 객체로 나누고, 이들의 상호작용을 서술하는 프로그래밍 패러다임. 재사용성, 유지보수성, 대규모 협업 시의 이점을 얻을 수 있다.
- **객체**: 현실에 물리적, 개념적으로 존재하는 모든 것
- **Class**: 관련된 속성과 메서드를 묶어 객체를 생성하기 위한 틀
- **객체지향의 특징**
    - **다형성**: 하나의 객체가 여러 형태를 가질 수 있는 성질
        - 오버로딩: 같은 이름, 다른 시그니처를 가지는 메서드를 정의하는 것
        - 오버라이딩: 부모 클래스의 메서드를 자식 클래스에서 재정의하는 것
    - **캡슐화**: 관련된 속성, 메서드를 묶고 내부 구현을 숨기는 것
    - **상속**: 자식 클래스가 부모 클래스의 속성, 메서드를 물려받아 재사용, 확장하는 것
    - **추상화**: 객체의 공통적이고 핵심적 특징을 파악해 정의하는 것 (추상 클래스, 인터페이스를 통해 구현)
- **상속과 인터페이스 차이**: 상속은 수직적 관계이며 IS-A , 인터페이스는 수평적 관계이며 CAN-DO를 의미

#### 제네릭과 컬렉션
- **제네릭**: 데이터 타입을 클래스나 메서드 사용 시점에 지정할 수 있도록 하는 문법
- **컬렉션**: 데이터를 효과적으로 처리하기 위해 자바에서 미리 만들어둔 표준화된 자료구조
- **대표적 컬렉션**
  - **List**: 순서 있는 배열
  - **Set**: 중복 없는 집합
  - **Map**: 키-값 쌍

#### 멀티스레드와 동시성
- **스레드**: 프로세스 내의 실행 흐름 단위. 동시성, 응답성을 위해 사용한다.
- **스레드 주의 사항**
  - **조용한 실패**: 스레드 내부에서 예외가 발생하면 조용히 종료되기 때문에, 명시적 예외 처리가 필요
  - **가시성 문제**: 여러 스레드가 동시에 자원에 접근 시 서로 다른 값을 볼 수 있음
  - **순서 문제**: 스레드 실행이 순차적으로 이루어지지 않음
- **Runnable**: 스레드 동작을 정의하는 표준 인터페이스로, 실행 로직과 제어를 분리
- **ExecutorService**: 스레드 풀 기반 비동기 작업 실행 관리 인터페이스. 자원 보호, 성능 향상, 관리 용이를 위해 사용한다.
- **동시성 제어 방법**
  - **Synchronized**: 공유 자원 접근을 하나로 제한하는 키워드. 실행 순서는 예측 불가능하다.
  - **Lock**: 멀티스레드 환경에서의 안전한 자원 접근을 보장하기 위한 동기화 제어 인터페이스. 안정성과 예측 가능성을 확보할 수 있다.
